# Datasets

This folder contains project datasets organized by dataset identity rather than by global `raw/`, `processed/`, and `metadata/` folders.

- `qsar_biodegradation_descriptor_benchmark/`: current working QSAR/QSPR descriptor benchmark used for reliability-aware biodegradation classification.
- `homopolymer_bigsmiles_tg_benchmark/`: Choi et al. (2024) homopolymer SMILES/BigSMILES benchmark with glass-transition-temperature labels; useful for polymer representation and property modeling, not biodegradation claims.
- `big_molecules_smiles_activity_benchmark/`: Kaggle SMILES activity dataset with `SMILES`, `pIC50`, `num_atoms`, and `logP`; useful for structure-aware ML prototypes but not biodegradation or polymer degradation claims.
- `bigsmiles_common_repeat_units_reference/`: planned BigSMILES notation/reference dataset from the Olsen Lab MIT BigSMILES repository. The full source content is not vendored until citation and license terms are verified.
- `polymer_structure_dataset_sources.md`: roadmap for finding future SMILES, BigSMILES, polymer-identity, and biodegradation datasets.

Each dataset should contain its own:

- `raw/`: original input data, kept as close to the source format as possible
- `processed/`: curated datasets generated by project scripts
- `metadata/`: dataset provenance, curation decisions, labels, and limitations

The current benchmark is descriptor-only. It does not contain polymer names, SMILES, BigSMILES, repeat units, molecular graphs, or continuous degradation-rate targets. Future polymer-structure datasets should be added as separate sibling folders instead of being mixed with the descriptor benchmark.
